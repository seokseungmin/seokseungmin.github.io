# 다양한 의존 객체 주입

## 생성자를 이용한 의존 객체 주입

applicationContext.xml에 빈 태그를 이용해서 객체를 생성하는데 studentDao라는 것을 생성합니다.<br>
그 다음에 레지스터 서비스를 빈 태그를 이용해서 또 생성을 합니다.<br>
생성할 때 생성자에다가 이렇게 studentDao객체를 주입하기 위해선 construcor argument라는 태그를 이용합니다.<br>
Reference 속성으로, 속성 값으로 주입하려는 객체의 id값(studentDao)을 넣어주면 Service 객체가 생성될때,<br>
그 안에 studentDao라는 객체가 주입될 수가 있다.
정리해서 생성자 주입하는 첫번째 방법, construcor argument라는 태그를 이용해서 해당하는 레퍼런스 속성의 속성값으로 해당하는 객체의 id값만 넣어주면 된다.<br>
그러면 객체가 스프링 컨테이너에서 생성이 될 때, 바로 주입이 되면서 생성이 된다.

## setter를 이용한 의존 객체 주입
property 태그를 이용하는데 이용할 때 다음 규칙을 지켜야함.<br>
속성값은 setter메서드의 이름에서 set을 떼고 첫번째 글짜 대문자를 소문자로 바꿔서 표현을 한다.<br>
value는 파라미터로 들어오는 값을 적어주면 됨.

## Map타입 객체 주입
맵 타입은 key,value로 이루어져 있음.<br>
administrators라는 setter 메서드가 데이터를 맵으로 받음.<br>
key, value 쌍을 이뤄서 줄수 있음.<br>
value에 해당하는 게 객체면,<br>
```xml
<map>
    <entry>
      <key>
        <value>dev</value>
      </key>
      <ref bean="dataBaseConnectionInfoDev"/>
    </entry>
    <entry>
      <key>
        <value>real</value>
      </key>
      <ref bean="dataBaseConnectionInfoReal"/>
    </entry>
</map>
```
이런식으로 주입. 밑에 전체 xml 코드를 참고하여 형태를 파악하고 이해해보자.
