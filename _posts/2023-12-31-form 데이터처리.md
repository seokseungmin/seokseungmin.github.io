```
form 태그의 대표적인 속성, 액션과 method가 있음.
method="get"방식 설정, submit 버튼을 누르면 나의 정보, 유저 데이터가 날아감.
서베에서 doGet() 메서드가 날린 데이터를 받음.

method="post"방식 설정, submit 버튼을 누르면 나의 정보, 유저 데이터가 날아감.
서베에서 doPost() 메서드가 날린 데이터를 받음.

왜 doGet, doPost로 받는 방식을 나눠 놨을까?
get방식은 데이터가 웹 브라우저 URL에 노출되어 웹 서버로 전송된다.
이게 무슨 얘기나면 우리가 보통 http한 다음에 도메인 주소 나오고 여기 포트 번호,
포트번호는 보통 우리가 네이버나 구글 이용할 때는 안 나오죠.
기본포트 80포트를 쓰고 있기 때문에 안나오는 건데요.
프로토콜이 나오고 도메인 주소 나오고 그 다음에 여러분들이 브라우저를 통해서 어떤
검색 사이트라든가 이런것을 해보시면 바로 알수가 있겠는데요.
이런 물음표가 나오고 여기에 Key, Value로 나옴. 내가 form태그에 입력한 정보가
이렇게 브라우저에 좀 지져분 하지만 쭉 해서 나열되서 서버로 날라가는 방식이 get방식.
이런 데이터를 누가 가로채서, 파싱을 해서 개인정보 등을 가로챌수 있음. 보안에 취약한 단점존재!
doPost방식은 굉장히 심플하게 URL이 날라감.
프로토콜 먼저 나오고 그  다음에 도메인 주소 나오고 Context Path까지 도메인 주소라고 생각해주면됨.
이 뒤에는 우리가 서버 맵핑을 하죠? 서블릿은 매핑해서 쓰죠. 그 맵핑 정보만 나와요.
만약에 그 서버 맵핑을 하고 있는 파일 Java파일이겠죠?
Java파일이 어떤 거인지를 알수가 없죠. 그래서 서블릿은 굉장히 보안에 강한데요.
이때, 이것만으로 강하다는게 아니라 사용자의 데이터가 헤더 파일에 암호화돼서 전송이 됩니다.
그렇기 때문에 보안에 굉장히 강한 효력을 발휘할 수가 있습니다.
그래서 데이터가 HTTP Request 헤더 파일안에 암호화돼서 웹서버 전송이 되기 때문에 굉장히 중요함.
Post방식으로 이용하는 예로는 로그인이라던가, 회원가입, 설문조사 등등 거의 대부분이 다 Post방식으로 전송이 된다고 볼수 있음.

참고 :  goGet 방식에서 정보를 보낼때, 한계가 있음. 많은 정보가 날아가는 경우는 많이 없겠지만,
텍스트가 나열될 수 있는 길이에도 한계가 있음. 그렇기에 너무 많은 데이터를 보내기에는 문제가 있음.
```

```java
package com.servlet;

import java.io.IOException;
import java.util.Arrays;
import java.util.Enumeration;

import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet("/mSignUp")
public class MemSignUp extends HttpServlet {

	protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
		
		System.out.println(" -- doGet() -- ");
		
		String m_name = request.getParameter("m_name");
		String m_pass = request.getParameter("m_pass");
		String m_gender = request.getParameter("m_gender");
		String[] m_hobbys = request.getParameterValues("m_hobby");
		String m_residence = request.getParameter("m_residence");
		
		System.out.println("m_name : " + m_name);
		System.out.println("m_pass : " + m_pass);
		System.out.println("m_gender : " + m_gender);
		System.out.println("m_hobbys : " + Arrays.toString(m_hobbys));
		System.out.println("m_residence : " + m_residence);
		
		Enumeration<String> names = request.getParameterNames();
		while (names.hasMoreElements()) {
			String name = (String) names.nextElement();
			System.out.println("name : " + name);
		}
		
	}

	protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
		
		System.out.println(" -- doPost() -- ");
		
		doGet(request, response);
	}

}
```
